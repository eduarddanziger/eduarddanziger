## Current work

I currently maintain the public **[collect-sound-devices](https://github.com/collect-sound-devices)** solution domain, a cross-platform system for monitoring, transporting, storing, and visualizing audio endpoint device data.

- **[win-sound-engine](https://github.com/collect-sound-devices/win-sound-engine)** native Windows audio engine in C/C++ with Go and C# frontends
- **[win-sound-scanner-go](https://github.com/collect-sound-devices/win-sound-scanner-go)** Windows audio device monitor in Go publishing events to RabbitMQ or Kafka
- **[linux-sound-scanner](https://github.com/collect-sound-devices/linux-sound-scanner)** Linux audio device scanner in C++
- **[audio-device-repo-server](https://github.com/collect-sound-devices/audio-device-repo-server)** ASP.NET Core REST API for the audio device repository
- **[list-audio-react-app](https://github.com/collect-sound-devices/list-audio-react-app)** React / Next.js / TypeScript web UI for the audio device repository
- **[kafka-to-rest-api-forwarder](https://github.com/collect-sound-devices/kafka-to-rest-api-forwarder)**, **[rmq-to-rest-api-forwarder](https://github.com/collect-sound-devices/rmq-to-rest-api-forwarder)** .NET microservices forwarding Kafka or RabbitMQ events to a REST API
